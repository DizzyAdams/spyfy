# 🎯 Visão & Missão — SpyFy

## Missão

> **Democratizar o acesso à inteligência de anúncios**, permitindo que qualquer profissional de marketing encontre, entenda e replique ofertas vencedoras em minutos — não em semanas.

## Visão de longo prazo

Tornar-se a **camada de inteligência competitiva padrão** do ecossistema de performance marketing global. Assim como o Ahrefs é para SEO e o Semrush é para marketing digital, o **SpyFy será o padrão de fato para ad intelligence e offer cloning**.

Até 2028, queremos:

- Indexar **1 bilhão+ de anúncios ativos** em tempo quase real.
- Cobrir **10+ redes** (Meta, TikTok, Google, YouTube, Native, Pinterest, LinkedIn, X, Snapchat, Kwai).
- Ser usado por **100.000+ media buyers, afiliados e agências**.
- Reconstruir automaticamente **funis completos** (LP → VSL → upsell → downsell) com um clique.

## O problema

Media buyers, afiliados e infoprodutores gastam **dezenas de horas por semana**:

1. Vasculhando manualmente bibliotecas de anúncios (Meta Ads Library, TikTok Creative Center).
2. Salvando prints e links soltos em pastas desorganizadas.
3. Tentando adivinhar **quais ofertas estão escalando** (sem dados de longevidade/volume).
4. Reconstruindo landing pages "no olho", perdendo detalhes de copy, estrutura e pixels.
5. Sem histórico: quando um anúncio sai do ar, a inteligência é perdida.

O resultado é **desperdício de budget** testando ofertas mortas e **atraso competitivo**.

## A solução

O SpyFy centraliza todo o ciclo de inteligência de ofertas:

```
Descobrir  →  Analisar  →  Clonar  →  Testar  →  Escalar
   │            │            │          │          │
 Discovery   Analytics    Cloner    Export     Alertas
```

### Diferenciais

| Diferencial | Descrição |
|-------------|-----------|
| **Longevidade real** | Rastreamos há quanto tempo cada anúncio está ativo — o principal sinal de oferta vencedora. |
| **Reconstrução de funil** | Não só a LP: mapeamos upsells, order bumps, thank-you pages. |
| **Transcrição de VSL** | Whisper + LLM transcreve e resume vídeos de vendas longos. |
| **Sub-agents de IA** | Agentes que raspam, classificam e reconstroem 24/7. |
| **Swipe file colaborativo** | Coleções compartilhadas com o time, tags e alertas. |
| **Histórico imutável** | Snapshots versionados — nada se perde. |

## Princípios de produto

1. **Velocidade acima de tudo** — inteligência que chega tarde é inútil.
2. **Dados > opinião** — decisões guiadas por longevidade e volume estimado, não achismo.
3. **Menos cliques** — do "quero ver ofertas de emagrecimento" ao insight em < 3 cliques.
4. **Ético e legal** — respeitamos ToS, LGPD/GDPR e direitos autorais (ver [compliance](../09-security/compliance.md)).
5. **Extensível** — API-first; tudo que a UI faz, a API também faz.

## North Star Metric

**Weekly Winning Offers Actioned (WWOA)** — número de ofertas vencedoras que um usuário efetivamente *salva, clona ou exporta* por semana. É o proxy direto de valor entregue.

## Métricas de suporte

- **Time-to-First-Insight (TTFI):** < 5 min do signup ao primeiro insight.
- **Offer Freshness:** % de ofertas atualizadas nas últimas 24h.
- **Clone Success Rate:** % de clonagens que reproduzem a LP com fidelidade > 95%.
- **Retention D30:** > 45%.

## Modelo de negócio

Assinatura SaaS por tiers (ver [product-overview](product-overview.md)) + créditos de clonagem + API enterprise.

## Por que agora?

- O custo de aquisição (CPM) subiu; **testar oferta certa** virou vantagem competitiva crítica.
- LLMs baratearam a **transcrição e reconstrução** automáticas.
- Bibliotecas públicas (Meta Ad Library, Google Transparency) criaram uma **base de dados legalmente acessível**.
- O mercado de spy tools legado (AdSpy, BigSpy) está **estagnado e caro**, com UX datada.
